1)FSV AND ITS TYPES:
FSP (Firmware Support ) is an Intel-provided binary firmware component that initializes silicon hardware (CPU, chipset, memory) during the early boot stages.
FSP executes the low-level CPU instructions that directly program hardware registers to make that hardware usable.
Why FSP exists
Modern CPUs and chipsets are extremely complex. Intel provides FSP so that:
•	Board/BIOS developers don’t need to write low-level silicon init code
•	Hardware initialization is consistent and validated
•	Time-to-market is faster
FSP is split into three logical parts:
1️⃣ FSP-T (Temporary RAM Init)
•	Sets up Cache-as-RAM
•	Allows PEI phase to run
•	Very first FSP entry point
2️⃣ FSP-M (Memory Init)
•	Initializes DRAM
•	Trains memory
•	Hands over real RAM to BIOS
3️⃣ FSP-S (Silicon Init)
•	Initializes remaining silicon:
o	PCIe
o	USB
o	Power management
o	SoC features
2. what is boot policy?
Boot Policy means the rules the firmware follows to decide how, from where, and in what order the system boots an OS. It is stored in BIOS/UEFI variables (NVRAM)and implemented by DXE + BDS (Boot Device Selection) phase.

1️⃣ Boot device order
•	SSD first, then USB, then Network
•	Legacy vs UEFI boot
•	Fallback if first device fails
2️⃣ Boot mode rules
•	UEFI boot only
•	Legacy (CSM) allowed or not
•	Mixed mode allowed
3️⃣ Security rules
•	Secure Boot enabled/disabled
•	Only signed OS loaders allowed
•	Block unknown USB boot

4. what’s ACPI?
ACPI stands for Advanced Configuration and Power Interface.
ACPI is a standard that tells the operating system how to manage power and hardware using information provided by the BIOS/UEFI.
Why ACPI exists
Before ACPI:
•	BIOS controlled power
•	OS had little control
With ACPI:
•	OS controls power
•	BIOS just describes the hardware and rules
So:
BIOS describes → OS decides → Hardware obeys
What ACPI does
1️⃣ Power management
•	Sleep states
•	Shutdown
•	Idle power saving
2️⃣ Device power control
•	Turn devices ON/OFF
•	Save power when idle
3️⃣ Hardware description
•	Tells OS:
o	Which devices exist
o	How they are connected
o	How to control them
ACPI power states (important)
System states    State	Meaning
S0	System ON
S3	Sleep (Suspend to RAM)
S4	Hibernate
S5	Soft OFF
Device states
•	D0 → Fully ON
•	D3 → OFF
5.what is a software hook?
A software hook is a mechanism that allows one piece of software to intercept, monitor, modify, or extend the behaviour of another piece of software at a specific point.
How a software hook works
At a defined point in execution:
1.	Program reaches a hook point
2.	Control is transferred to hook code
3.	Hook code runs (log, modify, block, extend)
4.	Control returns to original flow
In firmware:
A hook is a predefined callback that platform code can register to run at a specific boot or runtime stage.
Examples:
•	Before memory init
•	Before OS handoff
•	On S3 resume
•	On power button press

6.What is SMM?
System Management Mode (SMM) is a special, ultra-privileged CPU mode on x86 processors, designed for low-level system management tasks that must be:
•	Invisible to the OS
•	Isolated from normal software
•	Protected from malware
It was introduced by Intel for power and hardware management, but today it is the most security-critical execution environment in firmware.
SMM is entered only when an SMI occurs.
When an SMI fires:
1.	CPU immediately stops current execution
2.	CPU state is automatically saved
3.	CPU jumps to SMM code
4.	OS has zero awareness
 CPU State Is Saved To: SMRAM
•	All registers
•	Control registers
•	Segment registers
•	Instruction pointer
This saved context is called the SMI Save State.

7.What is CRC (Cyclic Redundancy Check)?
CRC (Cyclic Redundancy Check) is an error-detection technique used to verify the integrity of data—to check whether data was altered, corrupted, or damaged during:
•	Storage (flash, disk, memory)
•	Transmission (USB, PCIe, Ethernet)
•	Firmware updates (BIOS/UEFI images)
 CRC detects errors; it does NOT correct them.

Where CRC Is Used
•	Firmware Volume (FV) headers
•	Capsule update validation
•	NVRAM variable integrity
•	Option ROM validation

11.What is FSP?
FSP (Firmware Support Package) is a binary-only silicon initialization package provided by Intel.
It performs early CPU + chipset initialization, so OEMs don’t need to write or maintain complex silicon bring-up code.
 Think of FSP as:
“Intel initializes the silicon, BIOS focuses on platform policy.”

 What is FSP 2.0?
FSP 2.0 is the modern, standardized version of Intel FSP used in UEFI-compliant platforms.
It cleanly separates silicon initialization from platform firmware.
 What FSP 2.0 Does
FSP 2.0 initializes:
•	CPU cores
•	Memory (DDR training)
•	Chipset fabric
•	Power management
•	I/O controllers (basic setup)
And then returns control back to the BIOS.
FSP 2.0 Architecture
FSP 2.0 is divided into three binary components:
 FSP-T (TempRamInit)
•	Runs very early
•	Sets up temporary RAM (Cache-as-RAM)
•	No DRAM yet
 Used in SEC / early PEI
 FSP-M (MemoryInit)
•	Initializes DRAM
•	Performs memory training
•	Configures memory controller
 This is the most critical part
 FSP-S (SiliconInit)
•	Initializes remaining silicon
•	PCIe root ports
•	Integrated graphics
FSP-X is an extended / next-generation FSP model used in newer Intel platforms, especially those targeting:
•	Client SoCs
•	Hybrid architectures
•	Modular firmware stacks
 FSP-X is not a replacement — it is an evolution.
FSP-X Philosophy
FSP-X is designed for:
•	Complex SoCs
•	Multiple power domains
•	Fine-grained silicon control
•	Faster resume paths

19.What is LZMA Compression? 
LZMA (Lempel-Ziv-Markov chain Algorithm) is a high-ratio, lossless compression algorithm widely used in BIOS/UEFI firmware, bootloaders, and embedded systems.
Lossless → original data is recovered exactly
High compression → saves a lot of flash space
 Why LZMA Exists
Firmware storage (SPI flash) is very limited (8–32 MB).
LZMA allows:
•	Smaller firmware images
•	More features in the same flash
•	Faster boot reads (less data to fetch)
How LZMA Works
LZMA combines two techniques:
 LZ (Dictionary Compression)
•	Finds repeated byte sequences
•	Replaces them with references (distance + length)
 Markov Chain + Range Coding
•	Predicts probability of next bits
•	Encodes data efficiently using range encoding
 This is why LZMA compresses better than ZIP or GZIP, but needs more CPU.

26.What are mmio and page tables?
1. Memory-Mapped I/O (MMIO)
Definition
Memory-Mapped I/O (MMIO) is a mechanism where hardware device registers are mapped into the system’s physical memory address space. The CPU accesses these device registers using the same load and store instructions used for normal memory.
Purpose of MMIO
MMIO allows:
•	Uniform access to devices using standard memory instructions
•	Simplified CPU and compiler design
•	Faster device access compared to legacy I/O methods
•	Compatibility with modern 32-bit and 64-bit architectures
How MMIO Works
1.	A hardware device exposes control and status registers at fixed physical addresses.
2.	The operating system or firmware maps these addresses into its address space.
3.	Software reads or writes to these addresses.
4.	The hardware device responds to the access.
Page Tables
Definition: Page tables are data structures used by the CPU’s Memory Management Unit (MMU) to translate virtual addresses used by software into physical addresses used by hardware.
Why Page Tables Exist
Programs use virtual addresses
RAM uses physical addresses
The page table connects the two.
Virtual Address ──▶ Page Table ──▶ Physical Address

Purpose of Page Tables
Page tables provide:
•	Memory isolation between processes
•	Protection and access control
•	Support for virtual memory

28.What is micro code?
Microcode
1. Definition
Microcode is a low-level set of internal instructions stored inside a CPU that controls how machine-level instructions are executed by the processor.
It acts as an intermediate layer between:
•	Machine instructions (x86, ARM, etc.)
•	Physical CPU hardware (ALU, registers, pipelines)
Why Microcode Exists
Modern CPU instructions are complex and cannot always be executed directly by hardware in a single step. Microcode breaks these complex instructions into simpler internal operations (called micro-operations or µops).
Microcode allows:
•	Simplification of CPU hardware design
•	Support for complex instruction sets (e.g., x86)
•	Bug fixes after manufacturing
•	Behavioural consistency across CPU generations

32.What is s3 resume?
Definition
S3 Resume refers to the process of restoring a system from the S3 power state, also known as Suspend-to-RAM.
In the S3 state:
•	System RAM remains powered
•	CPU, chipset, and most devices are powered off
•	System context is preserved in memory
S3 Resume Process
1.	System enters S3 state on sleep request
2.	RAM contents are preserved
3.	Power is removed from CPU and devices
4.	On wake-up event (keyboard, mouse, timer):
o	CPU resets
o	Firmware performs minimal initialization
o	Memory contents are restored
o	OS resumes execution from saved state
33.What is watch dog timer and monotonic counter?
A Watchdog Timer (WDT) is a hardware or software timer used to detect and recover from system failures.
It monitors whether the system (CPU / firmware / OS) is operating correctly.
If the system fails to respond or gets stuck (for example due to a software hang or infinite loop), the watchdog timer automatically resets or recovers the system.
Why is a Watchdog Timer Needed?
In real-time and low-level systems such as:
•	BIOS / UEFI firmware
•	Embedded systems
•	Operating systems kernel
•	Industrial controllers
there may be no user interaction to recover the system when it hangs.
The watchdog timer ensures:
•	System reliability
•	Automatic recovery
•	Prevention of permanent system lock-ups
How Does a Watchdog Timer Work?
1.	The watchdog timer is started with a fixed timeout value
2.	The software must periodically “kick” or “pet” the watchdog
3.	If the software works correctly → watchdog keeps resetting
4.	If the software hangs or crashes → watchdog expires
5.	On expiry → system performs a reset, interrupt, or recovery action



34.what is FAT 32 and NTFS file system?
What is FAT32?
FAT32 (File Allocation Table 32) is an older and simple file system developed by Microsoft.
It is widely supported by almost all operating systems and devices.
Key Features of FAT32
•	Uses a File Allocation Table to track file locations
•	Simple structure
•	Highly compatible across platforms
•	No built-in security or permissions
FAT32 Limitations
•	Maximum single file size: 4 GB
•	Maximum partition size: ~2 TB
•	No file permissions
•	No encryption
•	No journaling (less crash-safe)
What is NTFS?
NTFS (New Technology File System) is a modern, advanced file system developed by Microsoft for Windows operating systems.
Key Features of NTFS
•	Supports very large files and partitions
•	Journaling for crash recovery
•	File permissions and access control
•	File encryption and compression
•	Better performance and reliability
NTFS Limits
•	Maximum file size: Up to 16 TB (practically much larger)
•	Maximum volume size: Up to 256 TB or more (depends on cluster size)

•	Generated on key press (make code) and key release (break code)
•	Same scan code regardless of keyboard language
•	Used by BIOS, firmware, and OS keyboard drivers
•	Example: Pressing the A key generates a scan code, regardless of language.
What is Unicode?
Unicode is a universal character encoding standard that represents characters and symbols from all languages.
•	Represents characters, not keys
•	Language and layout dependent
•	Used at application level
Key Characteristics of Unicode
•	Can represent characters from all languages
•	Same character has same Unicode value everywhere
•	Generated after OS layout and language processing
•	High-level representation
Example
•	Character ‘A’ → Unicode U+0041
•	Character ‘अ’ → Unicode U+0905
39.What is NV-RAM?
NV-RAM is a type of memory that retains stored data even when power is turned off, while still allowing random access like normal RAM.
In simple words:
NV-RAM combines the speed of RAM with the persistence of storage.
Why NV-RAM is Needed
Traditional RAM (DRAM):
•	Fast
•	❌ Loses data when power is off
Traditional Storage (HDD/SSD):
•	Non-volatile
•	❌ Slower than RAM
NV-RAM solves this gap by providing:
•	Fast access
•	Data persistence
Key Characteristics of NV-RAM
•	Non-volatile (data preserved without power)
•	Random access
•	Faster than disks
•	Used for critical system data
Common Types of NV-RAM
1. Flash Memory (Modern NV-RAM)
•	EEPROM / NOR / NAND flash
•	Used in:
o	BIOS / UEFI firmware
o	Embedded systems
Stores:
•	Firmware code
•	Configuration variables
2. NVRAM in UEFI
In UEFI systems:
•	NVRAM is a reserved region of flash
•	Stores:
o	Boot variables
o	Secure Boot keys
o	Boot order
o	Platform settings
Examples:
•	BootOrder
•	Boot####
•	SecureBoot
