STEP 0: What already exists before PciBusDxe

Before PciBusDxe does anything:

The hardware PCI slots already exist

The platform already says:

“Here is a PCI root bridge”

But no PCI devices are known yet

Think:

The building exists, but nobody has checked which rooms are inside.

STEP 1: “Can I manage this?” (Supported)

System asks PciBusDxe:

“Can you manage this PCI building?”

PciBusDxe checks only simple things:

Is this really a PCI root?

Is nobody else managing it?

If yes → “OK, I can manage it”

⚠ Nothing changes yet.

STEP 2: Start is called (ownership)

Now system says:

“You are the manager now.”

What Start really means:

PciBusDxe takes control

Only one manager is allowed

From now on, all PCI devices go through PciBusDxe

STEP 3: Look for devices (discovery)

PciBusDxe now looks everywhere:

Slot 0, slot 1, slot 2…

Function 0, function 1…

For each place it asks:

“Is anyone here?”

If no answer → empty
If answer → PCI device found

This is just finding, not using.

STEP 4: Understand what each device needs

For every found device, PciBusDxe asks:

“How much space do you need?”

Devices reply:

“I need memory”

“I need I/O space”

Still:

Nothing is enabled

Nothing is running

Just planning.

STEP 5: Give addresses (resource assignment)

Now PciBusDxe acts like an address allocator:

Gives each device:

A memory address

An I/O address

Makes sure:

No two devices conflict

This step is very important.

STEP 6: Turn devices ON

After addresses are assigned:

PciBusDxe tells each device:

“Here is your address, you may start working”

Now devices:

Can respond

Can transfer data

Before this step, they were silent.

STEP 7: Tell the rest of the system

Now PciBusDxe announces:

“These devices exist!”

For each device:

Creates a child

Other drivers can now see it

Example:

Network driver sees network card

Storage driver sees SSD

STEP 8: System is stable

Now everything is calm:

PCI devices are ready

Drivers are attached

PciBusDxe just watches quietly

This is the normal state.

What about Stop?

Stop is like saying:

“Manager, please leave the building.”

Why this almost never happens:

Devices are already in use

Removing them breaks the system

PCI was not designed to be removed in firmware

So Stop exists mainly:

For rules

For testing

For rare hot-plug cases

Super-simple summary (memorize this)
Start:
1. Take control
2. Find devices
3. Ask what they need
4. Give addresses
5. Turn them on
6. Tell other drivers

Stop:
- Try to undo this (if possible)

One sentence explanation (simplest)

PciBusDxe Start = discover → plan → assign → enable → publish
